# IT Asset Delivery System - Authentication Guide

## 🔐 System Overview

This system implements a role-based access control with two types of users:
- **Admin**: Full access to the system dashboard
- **User**: Limited access via token-based links

---

## 📋 Login Methods

### 1. Admin Login
**Username**: `admin`  
**Password**: `1234`

**Access Level**: Full system access including:
- Dashboard (overview statistics)
- Create new forms
- View form history
- Send links to users

**How to login**:
1. Go to `main.html` (login page)
2. Enter credentials in the "Admin Login" section
3. Click "เข้าสู่ระบบ Admin"
4. Will be redirected to `admin.html`

---

### 2. User Login via Token
Users can only access the system through a special token link generated by Admin.

**How it works**:
1. Admin creates a form and generates a unique token
2. Admin shares a link with the user (format: `user.html?token=<TOKEN>`)
3. User can either:
   - Click the link directly, OR
   - Copy the token and paste it in the login page under "User Access"
4. System verifies the token and allows access to that specific form

**Access Level**: Limited to:
- View assigned form only
- Fill in checklist items
- Add notes and signature
- Submit form for review

---

## 🔑 Session Management

All sessions are stored in the browser's `localStorage`:
- **Key**: `auth_session`
- **Contains**: User role, username, and login time

### Session Data Structure
```json
{
  "role": "admin",
  "username": "admin",
  "loginTime": "2026-02-27T10:30:00.000Z"
}
```

---

## 🛡️ Security Features

1. **Protected Routes**
   - Non-logged-in users cannot access admin.html
   - Users must have a valid token to access user.html

2. **Session Verification**
   - Each page checks if user has valid session
   - Invalid sessions redirect to login page

3. **Token-Based User Access**
   - Each user token is unique per form
   - Tokens are validated against the database

4. **Logout Functionality**
   - Users can logout from their respective pages
   - Clears session and redirects to login page

   - Either click the link directly
   - Or go to login page and paste the token

## 💾 Data Storage

All form data is stored in browser's `localStorage`:
- **Key**: `it_forms`
- **Format**: JSON array of form objects
- **Persistence**: Data persists until browser cache is cleared

### Form Object Structure
```javascript
{
  id: "unique-id",
  token: "user-access-token",
  status: "draft|sent|user_filled|completed",
  empName: "Employee Name",
  empCode: "EMP001",
  assetCode: "AST001",
  checklist: [...],
  adminSig: "signature-data",
  userSig: "signature-data",
  adminNote: "Admin notes",
  userNote: "User notes",
  createdAt: "2026-02-27T10:00:00Z",
  updatedAt: "2026-02-27T10:30:00Z"
}
```

## 🔄 User Flow

### Admin Flow
```
main.html (Login) 
  ↓
admin.html (Dashboard)
  ├→ 📊 View Dashboard (statistics)
  ├→ ➕ Create Form (new form)
  └→ 📋 View History (all forms)
     └→ Generate token & share link with user
```

### User Flow
```
Link from Admin / Login Page
  ↓
user.html?token=XXX (Verify token)
  ✓ Token valid → Show form
  ✗ Token invalid → Show error
  ↓
Fill checklist & sign
  ↓
Submit form
  ↓
user.html?token=XXX (Show confirmation)
```

## 🔗 API Methods (static/js/auth.js)

### Authentication Methods
- `Auth.loginAdmin(event)` - Admin login
- `Auth.loginUser()` - User login via token
- `Auth.logout()` - Logout current session

### Session Methods
- `Auth.getCurrentSession()` - Get current session data
- `Auth.isLoggedIn()` - Check if logged in
- `Auth.isAdmin()` - Check if user is admin

## 📝 Notes

- All times are in ISO 8601 format (UTC)
- Forms can only be accessed by the assigned user via token
- Logout is available from admin and user pages
- Session data is stored per browser/device

## 🐛 Troubleshooting

| Issue | Solution |
|-------|----------|
| Can't login as admin | Check username=`admin`, password=`1234` |
| User link doesn't work | Make sure token is copied correctly (case-sensitive) |
| Session expired | Close and reopen the page, or logout and login again |
| localStorage full | Clear browser cache/cookies and reload |
